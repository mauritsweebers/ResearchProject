PURPOSE

This protocol governs how Command:
- Issues module directives
- Audits and sync-checks outputs
- Executes quality assurance (QA)
- Closes modules and phases

It supersedes all earlier versions.

------------------------------------------------------------
ACTIVATION

This protocol is active across all project phases. Triggered by:
1. Module completion declaration
2. Metadata submission
3. Phase completion request

------------------------------------------------------------

Command initiates each module. Directives include:
- Module number and scope
- Required outputs (DT, TX, FG)
- Copy-ready prompt for Phase Chat
- SSOT file structure and naming instructions
