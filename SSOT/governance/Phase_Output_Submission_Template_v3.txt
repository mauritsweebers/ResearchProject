Phase Output Submission Template
Issued by: Project Command
Purpose: To standardize indexing and traceability of all phase outputs.

------------------------------------------------------------
📤 Submission Format (to be posted in Command Chat):

Index Code (Provisional): P[X].M[Y].[TYPE][##]
Phase: X
Module: Y
Type: [DT = Data Table | FG = Figure | TX = Text Output | RS = Research Summary | SC = Scenario | OT = Other]
Title/Description: [Concise, descriptive name of output]
File Name: [e.g., P0_M1_DT01_GlobalFoodProduction.csv]
Storage Location: /ssot/phase[X]/module[Y]/[subfolder]/
Source Category (if data-driven): [🟢 A | 🟡 B | 🔴 C]
Notes: [Optional – methods used, gaps, prompt summary, references cited]

------------------------------------------------------------
📍 Example:

Index Code (Provisional): P0.M1.DT01
Phase: 0
Module: 1
Type: DT
Title/Description: Global food production by category (weight + calories)
File Name: P0_M1_DT01_GlobalFoodProduction.csv
Storage Location: /ssot/phase0/module1/outputs/
Source Category: 🟢 A
Notes: Based on FAOSTAT 2022 + crosschecked with WRI. Includes feed allocation estimates.

------------------------------------------------------------
📁 DIRECTORY REFERENCE GUIDE

Use the correct subfolder for each file type:

/data/        → Raw or processed data tables (CSV, XLSX, JSON)
/outputs/     → Generated text, visuals, and derived analytical products (TXT, MD, SVG, PNG)
/references/  → Source documents, literature, citations (PDF, RIS, BibTeX)
/logs/        → Prompt-response records, analyst notes, method drafts

Example: /ssot/phase1/module2/data/P1_M2_DT02_YieldEstimates.csv

All files must be stored according to this structure and submitted for indexing.
